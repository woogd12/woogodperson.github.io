### 082의 문제 해답 및 사유를 작성

# ROLE 관련

The SET ROLE statement enables as well as disables roles in a session. (True - 정답)
해석: SET ROLE 문은 세션에서 역할을 활성화할 뿐만 아니라 비활성화하기도 한다.

정답인 이유:

SET ROLE 명령어는 단순히 롤을 켜는(Enable) 기능만 하는 것이 아니라, 현재 세션의 롤 상태를 재정의하는 명령어입니다.

예를 들어, SET ROLE role_A;를 실행하면 role_A는 활성화되지만, 그 외 다른 모든 롤은 자동으로 비활성화됩니다.

또한 SET ROLE NONE;을 실행하면 현재 세션의 모든 롤이 비활성화됩니다. 즉, 활성화와 비활성화 기능을 모두 수행합니다.

The ALTER USER statement must be used to set specific default roles for a user. (True - 정답)
해석: 사용자의 특정 기본 롤(Default Role)을 설정하려면 ALTER USER 문을 사용해야 한다.

정답인 이유:

사용자가 로그인할 때 자동으로 활성화될 롤을 지정하는 유일한 명령어는 ALTER USER입니다.

문법: ALTER USER 사용자명 DEFAULT ROLE 롤이름1, 롤이름2; 또는 ALTER USER 사용자명 DEFAULT ROLE ALL EXCEPT 롤이름;

이 설정을 통해 로그인 시 불필요한 권한이 과도하게 부여되는 것을 방지할 수 있습니다.
